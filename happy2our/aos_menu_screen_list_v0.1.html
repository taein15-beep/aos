01 AOS 관리자 메뉴구조 v0.1

02 대메뉴별 목적 요약
# 대메뉴 목적 주요 연결 도메인 핵심 이용자
1 상품관리 여행 상품의 전체 생명주기(등록·수정·공개·종료)를 관리. 일정표·요금·이미지·판매조건을 일괄 설정 예약, 판매사, 정산, 홈페이지 노출 상품팀 담당자
2 예약관리 인입된 예약의 상태 흐름(접수→확정→출발→완료)을 관리하고, 좌석·행사·출발 여부를 통제 상품, 고객, 정산, 메시지 예약팀, CS팀
3 정산관리 판매사·공급사와의 인보이스 발행 및 수수료 정산. 미지급 항목 추적 및 결재 이력 관리 예약, 판매사, 고객 정산팀, 경영진
4 실적관리 기간별·채널별·상품별 예약 및 매출 통계를 시각화. 수익성 분석 및 의사결정 지원 예약, 정산, 상품 경영진, 마케팅팀
5 메시지센터 예약 확정·취소·출발안내 등 여행 생애주기 알림을 카카오·SMS·이메일로 자동/수동 발송 예약, 고객, 상품 CS팀, 마케팅팀
6 회원관리 일반 고객·판매사·지자체·내부 직원 계정을 통합 관리. 접근 권한 그룹을 설정하여 역할 기반 통제 예약, 정산, 상품, 메시지 시스템 관리자
7 디자인설정 여행사 홈페이지의 페이지 구성·배너·GNB·팝업을 코드 없이 편집. 템플릿 기반 시각 설정 홈페이지 노출, 상품 마케팅팀, 운영자
8 기본설정 사이트 운영에 필요한 기초 정보(사업자·예약규정·노선·탑승역)를 설정. 전체 시스템 동작의 기준값 전 영역 시스템 관리자

04 지자체관리 메뉴 목적 및 기본 기능
지자체관리는 지방자치단체가 운영하는 여행 지원사업(관광 바우처, 여행 보조금, 지역 주민 우대 등)과 AOS 플랫폼의 예약·정산을 연결하기 위한 메뉴입니다. 여행사는 지자체와 계약을 맺고, 해당 지자체 주민 또는 사업 참여자에게 할인·보조금 혜택을 제공하며, 이를 별도 정산합니다.

🏛 메뉴 목적

  • 지자체 계정 및 계약 정보 등록/관리
  • 지자체별 지원사업(바우처, 보조금 등) 등록
  • 지원 대상 예약에 자동 할인 적용
  • 지자체 전용 정산서 별도 생성·관리
  • 지원 실적 보고용 통계 제공

🔗 연결 도메인

  • 예약관리 → 지원 예약 식별 및 할인 적용
  • 정산관리 → 지자체 별도 인보이스 발행
  • 상품관리 → 지자체 전용 상품/공개 조건
  • 메시지센터 → 지원 확정 안내 자동 발송
  • 실적관리 → 사업별 실적·예산 집행 현황

06 메뉴구조 보완 검토
검토 항목현황검토 의견권고 조치판정
상품공유관리 필요 여부 현재 없음 공급사가 상품을 여러 판매사와 공유하는 B2B 유통 구조 존재. 상품기본설정 내 판매사설정 탭만으로는 대량 공유 시 관리 한계 P3으로 추가. 판매사설정 탭 확장 후 별도 화면으로 분리 검토 추가 권장
예약상세 화면 필요 여부 현재 없음 (목록만 존재) 예약관리에서 목록 클릭 시 상세 화면이 없으면 CS 업무 불가. 예약자·탑승자·결제내역·상태이력 확인이 핵심 업무 흐름 P1으로 반드시 추가. 슬라이드오버 패널 또는 별도 페이지로 구현 필수 추가
지자체관리 확장 가능성 신규 메뉴로 추가 예정 지자체 포털 계정 분리, 공공 API 연동, 다중 사업 관리로 확장 가능. 초기에는 단순 등록+정산 구조로 시작해 점진 확장 권고 P2 구현 후 포털 분리는 P3으로 계획 단계 확장
메시지센터 구조 보완 4개 채널 분리 ①자동발송 트리거 설정 화면 없음 (예약완료 시 알림톡 자동발송 등) ②발송 실패 재발송 처리 흐름 없음 ③수신거부 고객 관리 화면 없음 자동발송 설정 탭 추가(P2). 수신거부 관리는 고객관리 내 필드로 처리 가능 보완 필요
상태값 관리 방식 화면별 산재 예약상태·상품상태·정산상태·발송상태 등 시스템 전반의 상태값이 문서화되지 않음. 개발 시 불일치 위험 별도 "상태값 정의서" 문서 작성 권고 (개발 착수 전 필수) 문서화 필요
공급사 관리 메뉴 현재 없음 판매사(판매채널)와 공급사(상품 제공처: 버스사, 숙박업체 등)는 역할이 다름. 공급사 계약·단가 관리가 정산과 연결되어야 함 회원관리 하위에 "공급사관리" 추가 검토(P2) 추가 검토
대시보드(메인화면) 현재 없음 로그인 후 첫 화면. 오늘의 예약현황·출발확정 대기·미정산 건 등 핵심 지표를 한눈에 파악하는 홈 대시보드 필요 P1으로 추가. 별도 대메뉴보다 로그인 후 기본 화면으로 구현 권장 필수 추가
